fix: robust handling for SMTC AppUserModelId media clicks

- Replaced basic ShellExecuteW with ShellExecuteExW using SEE_MASK_FLAG_NO_UI to suppress native Windows error dialogs on failed launches.
- Added robust path parsing to handle absolute paths injected directly by SMTC (common for desktop browsers like Chrome and Brave).
- Implemented heuristic to intelligently strip ' (x86)' from dead paths, automatically locating the executable if the user upgraded from a 32-bit to a 64-bit browser installation.
- Gracefully abort SMTC launches if the underlying executable truly no longer exists, preventing broken Windows App Paths registry keys from triggering 'The specified path does not exist' dialogs when clicking fading media sessions of closed browsers.
